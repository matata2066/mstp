# MSTP - UX 界面设计

## 1. 设计原则

- 采用企业级中后台设计语言，界面简洁专业
- 响应式布局，适配 1920x1080 及以上分辨率
- 统一使用 Ant Design Pro 组件库
- 所有操作需有明确反馈（成功/失败/加载中）
- 敏感信息脱敏显示
- 操作留痕，关键操作二次确认

## 8. 交互规范

### 8.1 操作反馈

| 场景 | 反馈方式 |
|------|----------|
| 表单提交成功 | 全局提示 Success，自动关闭弹窗，刷新列表 |
| 表单提交失败 | 全局提示 Error，保留弹窗，显示错误信息 |
| 表单校验失败 | 字段下方红色提示文字 |
| 列表加载中 | 表格 Loading 遮罩 |
| 审批通过/拒绝 | 二次确认弹窗 → 操作成功提示 |
| 删除操作 | 二次确认弹窗 |

### 8.2 数据脱敏规则

| 数据类型 | 脱敏规则 | 示例 |
|----------|----------|------|
| 银行账号 | 前4后4，中间* | 6225\*\*\*\*\*\*\*6789 |
| 账户名称 | 首字+* | 张\* |
| 行号 | 不脱敏 | 102100099996 |
| 金额 | 不脱敏 | 100,000.00 |

### 8.3 列表默认排序

- 支付指令：按创建时间倒序
- 审批记录：按制单时间倒序
- 操作日志：按操作时间倒序
